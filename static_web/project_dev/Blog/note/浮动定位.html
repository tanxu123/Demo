1.清除浮动

    1. 由于浮动，不在占据文档流的位置，通过clear可以解决页面排版的问题
h2{
    clear:left;/*清除左边浮动造成的影响*/
    clear:right;/*清除右边浮动造成的影响*/
    clear:both;/*同时清除两边的浮动造成的影响*/
    /*清除浮动的影响的使用元素一般是同级的元素，使用浮动清除，就等于该控制又回到了当初的文档控制流，会占据位置*/
}

      2.运用overflow属性清除浮动

     运用clear属性只能清除元素左右两侧浮动的影响，然而在网页中，如果对子元素设置浮动，不对父元素东一高度，则子元素的浮动会对父元素产生影响

  由于受到子元素浮动的影响，没有设置高度的父集会变成一条直线，即父元素不能自适应子元素的高度，这式对父元素设置overflow:hidden样式，既可以清除浮动对该元素的影响

h3{
    overflow:hidden;
}

       3.使用after伪对象清除浮动

    1.        设置伪对象的height:0;
    2. 在伪对象里面设置content属性，属性值可以为空  content=“”;
div{
    display:block;/*这个属性必须要进行设置，原因是如果不设置，由于内部的元素使用浮动脱离文档流，造成该div内部没有元素进行填充
    所以高度为0，显示为一条线*/
    clear:both;/*清除浮动的影响*/
    content:"";/**设置内容的值为空**/
    height:0;
    visibility:hidden;
}

值描述
visible默认值。内容不会被修剪，会呈现在元素框之外。

3.定位

position 属性指定了元素的定位类型。

position 属性的五个值：

static                 自动定位，默认

静态定位就是采用文档的默认定位模式，无法通过left，right，top，bottom来改变元素的位置

relative              相对定位，相对其原文档流的位置进行定位,需要注意的是会占据文档流

相对标准文档流的位置进行定位，也就是相对其在static定位的位置的偏移

fixed                   固定定位 相对浏览器窗口进行定位

会脱离文档流，不会随着浏览器窗口大小的改变来改变位置，一般很少使用.因为相对的是浏览器，当我们的文档过长时，显示的位置随着我们的滚动却没有变化，这是比较坑的。

absolute              绝对定位 相对上一个已经定位的父元素进行定位

依据最近的已经定位(绝对定位，固定定位，或相对定位)的父元素进行定位，如没有父元素进行定位，则依据body进行(浏览器窗口进行定位)

如果对元素设置绝对定位，但是不设置偏移量，元素的位置不变，但是会脱离文档流，定义多个边偏移属性时，如果left和right冲突使用left，如果

top与bottom冲突，使用top

sticky                   基于用户的滚动位置来定位。这个可以用来制作不能滚动的标题栏

 

元素可以使用的顶部，底部，左侧和右侧属性定位。然而，这些属性无法工作，除非是先设定position属性。他们也有不同的工作方式，这取决于定位方法。

如果我们需要设置子元素在父元素的直接位置保持不变，就可以设置父元素采用相对定位，但是不设置偏移量，然后设置子元素采用绝对定位
